# FoodieHub 数据库表结构设计文档

本文档详细描述了 FoodieHub 项目中所有数据表的结构及其相互关系。

## 1. Users 表 - 用户表

| 字段名     | 类型         | 属性          | 说明                                        |
| ---------- | ------------ | ------------- | ------------------------------------------- |
| id         | INT          | 主键，自增    | 用户唯一标识                                |
| username   | VARCHAR(50)  | 非空，唯一    | 登录用户名                                  |
| password   | VARCHAR(255) | 非空          | 加密后的密码                                |
| phone      | VARCHAR(20)  | 非空          | 手机号                                      |
| email      | VARCHAR(100) | 非空          | 电子邮箱，用于通知                          |
| avatar     | VARCHAR(255) | 可选          | 头像图片URL                                 |
| bio        | TEXT         | 可选          | 个人简介                                    |
| role       | INT          | 非空，默认0   | 角色：0:user（普通用户）、1:admin（管理员） |
| is_active  | BOOLEAN      | 默认TRUE      | 是否启用（软删除）                          |
| created_at | DATETIME     | 非空          | 注册时间                                    |
| updated_at | DATETIME     | 非空          | 最后更新时间                                |

## 2. Shops 表 - 店铺表

| 字段名         | 类型         | 属性       | 说明                                         |
| -------------- | ------------ | ---------- | -------------------------------------------- |
| id             | INT          | 主键，自增 | 店铺唯一标识                                 |
| name           | VARCHAR(100) | 非空       | 店铺名称                                     |
| description    | TEXT         | 可选       | 店铺描述                                     |
| campus_status  | VARCHAR(20)  | 可选       | 校园状态：on_campus(校内)/off_campus(校外)/unknown(未知)                                     |
| view_count     | INT          | 默认0      | 总浏览量（冗余字段）                         |
| favorite_count | INT          | 默认0      | 收藏数（冗余字段）                           |
| comment_count  | INT          | 默认0      | 讨论数（冗余字段）                           |
| average_rating | DECIMAL(2,1) | 默认0.0    | 平均评分（冗余字段）                         |
| category       | INT          | 可选       | 店铺类别ID（可能需要关联表）                 |
| is_active      | BOOLEAN      | 默认TRUE   | 是否启用（软删除）                           |
| created_at     | DATETIME     | 非空       | 创建时间                                     |
| updated_at     | DATETIME     | 非空       | 最后更新时间                                 |

| ----------- | ------------ | ---------- | ---------------------------- |

| sort_order  | INT          | 默认0      | 排序顺序                     |

| created_at  | DATETIME     | 非空       | 创建时间                     |
| updated_at  | DATETIME     | 非空       | 最后更新时间                 |

## 4. DiningMethods 表 - 点餐方式表

| ----------- | ------------ | ---------- | ---------------------------------------- |
| id          | INT          | 主键，自增 | 唯一标识                                 |

| updated_at  | DATETIME     | 非空       | 最后更新时间                             |

## 5. Ratings 表 - 评分表

| 字段名     | 类型     | 属性                | 说明                             |
| ---------- | -------- | ------------------- | -------------------------------- |
| id         | INT      | 主键，自增          | 评分唯一标识                     |
| user_id    | INT      | 外键，引用users(id) | 评分用户                         |
| shop_id    | INT      | 外键，引用shops(id) | 被评店铺                         |
| score      | INT      | 非空                | 评分值                           |
| is_active  | BOOLEAN  | 默认TRUE            | 是否启用（软删除）               |

| updated_at | DATETIME | 非空                | 最后更新时间                     |
| 唯一约束   | -        | (user_id, shop_id)  | 保证一个用户对一个店铺只能评一次 |

## 6. Comments 表 - 评论表

| 字段名      | 类型       | 属性                                   | 说明                                                     |
| ----------- | ---------- | -------------------------------------- | -------------------------------------------------------- |
| id          | BIGINT     | 主键，自增                             | 评论唯一标识                                             |
| shop_id     | INT        | 外键，引用shops(id)                    | 关联的店铺                                               |
| user_id     | INT        | 外键，引用users(id)                    | 发表评论的用户                                           |
| parent_id   | BIGINT     | 可为空，外键引用comments(id)           | 父评论。NULL表示一级评论；非空表示对该评论的回复         |

| content     | TEXT       | 非空                                   | 评论内容，支持富文本或纯文本                             |
| like_count  | INT        | 非空，默认0                            | 点赞数，冗余字段，便于排序和展示                         |
| reply_count | INT        | 非空，默认0                            | 直接子回复数量，冗余字段，减少COUNT查询                  |
| is_active   | BOOLEAN    | 默认TRUE                               | 是否启用（软删除）                                       |
| created_at  | DATETIME   | 非空                                   | 创建时间                                                 |
| updated_at  | DATETIME   | 非空                                   | 最后更新时间                                             |

## 表之间的关系

### 一对一关系
- 无

### 一对多关系
1. **Users → Activities**: 一个用户可以产生多个动态
2. **Users → Comments**: 一个用户可以发表多个评论
3. **Users → Ratings**: 一个用户可以对多个店铺进行评分
4. **Users → Favorites**: 一个用户可以收藏多个店铺
5. **Users → Messages**: 一个用户可以收到/发送多条消息
6. **Users → ShopEditRequests**: 一个用户可以提交多个店铺编辑申请
7. **Users → UserBehaviorLogs**: 一个用户可以有多条行为日志
8. **Shops → Comments**: 一个店铺可以收到多个评论
9. **Shops → Ratings**: 一个店铺可以收到多个评分
10. **Shops → Favorites**: 一个店铺可以被多个用户收藏
11. **Shops → MenuItems**: 一个店铺可以有多个菜单项
12. **Shops → ShopEditRequests**: 一个店铺可以有多个编辑申请
13. **Categories → ShopCategories**: 一个类别可以关联多个店铺
14. **DiningMethods → ShopDiningMethods**: 一个点餐方式可以被多个店铺使用
15. **Comments → Comments**: 一个评论可以有多个回复（自关联）

### 多对多关系
1. **Shops ↔ Categories**: 通过ShopCategories中间表实现多对多关系
2. **Shops ↔ DiningMethods**: 通过ShopDiningMethods中间表实现多对多关系
3. **Users ↔ Comments**: 通过CommentsLikes中间表实现用户对评论的点赞关系

### 特殊关系
1. **多态关联**:
   - **Images** 表通过 `entity_type` 和 `entity_id` 字段可以关联到多种不同的实体（shops, menu_items, comments等）
   - **Activities** 表通过 `target_type` 和 `target_id` 字段可以关联到多种不同的实体
   - **UserBehaviorLogs** 表通过 `target_type` 和 `target_id` 字段可以记录对多种不同实体的行为

2. **自关联**:
   - **Comments** 表通过 `parent_id` 和 `root_id` 实现评论的层级结构
