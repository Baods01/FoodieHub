# 01 — 项目理解

> 面向 Claude Code 的项目全景。开发者在理解需求时也可参考。

---

## 项目定位

食探社是一个**去商业化的华农校园美食互助社区**。解决两个核心矛盾：
- **信息有效性失效**：大众点评覆盖全城，数据颗粒度太粗，学生想"看西门糖水店评价"却混进商圈推荐
- **信息真实性失效**：刷评/恶评灰色产业链让评分公信力坍塌

**核心解法**：用户范围锁定华农校园步行可达区域 → 信息天然经地域筛选 + 小群体道德约束抑制刷评 → 用社区信任替代算法排名，用户共建替代商家入驻。

## 用户角色

| 角色 | 权限 | 设计约束 |
|:---|:---|:---|
| **游客** | 读全部（浏览、搜索、看评论/评分） | 写操作（收藏、评论、点赞）触发登录引导弹窗，不强制注册 |
| **普通用户** | 全部读 + 写操作 | 唯一内容生产者；操作需乐观更新（收藏/点赞即时反馈） |
| **管理员** | 全部权限 + 后台治理 | 零代码、低频（每日1-2次集中处理举报/勘误） |

共性特征：**高认知、高道德、低时间容忍度**。饭点前30分钟浏览高峰，饭后1小时内容生产高峰。

## 功能模块

| 编号 | 模块 | 核心页面 |
|:---|:---|:---|
| FD-01 | 轻量级店铺信息管理 | 上传店铺页、图片/菜单上传、勘误/重复反馈 |
| FD-02 | 点评与问答互动 | 店铺详情页（评论+问答+评分）、点赞组件 |
| FD-03 | 用户账户与收藏管理 | 登录/注册、个人中心、收藏夹、消息中心、浏览历史 |
| FD-04 | 店铺发现与检索 | 首页、搜索、筛选、排序、店铺详情 |
| FD-05 | 社区反馈与举报 | 举报弹窗（评论/店铺/图片） |
| FD-06 | 管理员后台治理 | 仪表盘、举报处理、店铺合并、封禁/解封、公告发布 |

## 系统边界

| 不做什么 | 原因 |
|:---|:---|
| ❌ 地图服务 | 店铺地址纯文本，无地理坐标 |
| ❌ 在线支付/外卖 | 纯信息分享平台 |
| ❌ 第三方登录 | 自建邮箱/手机号 + JWT 体系 |
| ❌ 短信验证码 | 课程设计复杂度控制 |
| ❌ 外部 API 调用 | 完全自包含，单服务器部署 |

## 设计原则

- **低摩擦分享**：店铺创建仅三必填字段（名称、区域、品类），即刻上线无需审核
- **社区自净**：评论/问答自然补全信息，举报触发管理员后置仲裁
- **乐观更新**：收藏、点赞先改 UI 再异步确认，失败回滚
- **游客友好**：读全开放，写拦截引导，不强制注册
