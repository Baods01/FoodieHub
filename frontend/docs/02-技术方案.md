## 选型总览

| 层面 | 方案 | 状态 | 理由 |

| 样式 | Tailwind CSS v3 | ✅ | 原子化 CSS，生产包 <10KB，响应式断点内置 |

| Toast | react-hot-toast | ✅ | 5KB，API 极简 |
| 全局状态 | zustand | ✅ | 无 Provider 嵌套，~1KB |
| 工具 | clsx | ✅ | 条件拼接 className |
| **服务端状态** | **@tanstack/react-query** | **⏳** | 缓存/乐观更新/重试 —— 后续 Phase 5 安装 |

## 关键架构决策

### 状态管理分层

```
TanStack Query (待安装)    → 服务端数据（店铺列表、评论、通知、用户资料）
Zustand (已安装)           → 纯客户端状态（搜索历史、UI 偏好）
useReducer (HomePage 已用)  → 页面级复杂状态（筛选条件组合）
```

### 乐观更新模式（收藏/点赞操作的标准流程）

```
用户点击 → onMutate: 快照缓存 + 立即切换 UI 状态
        → 异步请求
        → onError: 回滚缓存 + Toast "操作失败，请重试"
        → onSettled: 重新拉取最新数据（确保一致）
```

### 游客拦截模式

```
路由级拦截（AuthGuard）→ /favorites, /upload, /notifications 等页面
操作级拦截（Modal）    → 游客在详情页点"收藏/评论"弹出登录引导弹窗
```

### API 调用方式

所有 `api/*.ts` 文件中的函数先调用 mock 数据实现，真实接口签名以注释保留：

```typescript
export async function fetchShops(filter: ShopFilter) {
  // 后端对接时取消注释以下代码，删除 mockFetchShops 调用:
  // const params = { ...filter, page_size: 12 };
  // const { data } = await apiClient.get('/shops', { params });
  // return data.data;
  return mockFetchShops(filter);
}
```
