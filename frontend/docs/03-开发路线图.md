## 后端接口依赖时序

| 批次 | 接口 | 对应 Phase |
|:---|:---|:---|
| A | `GET /api/shops` `GET /api/shops/search` `GET /api/dictionaries/*` `GET /api/announcements/active` | Phase 2 ✅ mock 已完成 |
| B | `GET /api/shops/:id` `GET /api/shops/:id/comments` `GET /api/shops/:id/questions` | Phase 3 |
| C | `POST /api/auth/register` `POST /api/auth/login` | Phase 4 |
| D | `POST /api/shops/:id/rate` `POST /api/shops/:id/comments` `POST /api/comments/:id/like` `POST /api/shops/:id/favorite` 等 | Phase 5 |
| E | `GET/PUT /api/users/me` `GET /api/favorites` `GET /api/history` `GET /api/notifications` `POST /api/shops` 等 | Phase 6 |
| F | `POST /api/reports` `GET /api/admin/*` `POST /api/admin/*` 等 | Phase 7 |

> **对联调建议**：每个 Phase 开始前先用 Swagger (`/docs`) 确认该批次接口可用。后端未就绪时前端用 mock 数据先行开发 UI。
