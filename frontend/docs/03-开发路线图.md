## 阶段进度

| 阶段 | 内容 | 状态 |
|:---|:---|:---|
| Phase 1 | 项目地基（依赖安装 + Tailwind 配置 + 目录结构） | ✅ |
| Phase 2 | 主页（店铺列表 + 搜索/筛选/排序 + 卡片 + 骨架屏/空态/错误态） | ✅ |
| Phase 3 | 店铺详情页（轮播图 + 评分交互 + 评论区 + 问答 + 菜单 + 相册） | ✅ |
| Phase 4 | 用户认证（登录/注册 + 游客拦截 + JWT 管理） | 🔄 |
| Phase 5 | 互动操作（评论/评分/点赞/收藏 + 乐观更新） | 🔄 |
| Phase 6 | 个人中心 + 店铺上传（资料/收藏/历史/通知 + 创建店铺） | ⬜ |
| Phase 7 | 管理后台 + 举报反馈 | ⬜ |

## 后端接口依赖时序

| 批次 | 接口 | 对应 Phase |
|:---|:---|:---|
| A | `GET /api/shops` `GET /api/shops/search` `GET /api/dictionaries/*` `GET /api/announcements/active` | Phase 2 ✅ mock 已完成 |
| B | `GET /api/shops/:id` `GET /api/shops/:id/comments` `GET /api/shops/:id/questions` | Phase 3 ✅ mock 已完成 |
| C | `POST /api/auth/register` `POST /api/auth/login` | Phase 4 🔄 authStore 已就绪，登录/注册页面待开发 |
| D | `POST /api/shops/:id/rate` `POST /api/shops/:id/comments` `POST /api/comments/:id/like` `POST /api/shops/:id/favorite` 等 | Phase 5 🔄 LikeButton 组件已就绪，乐观更新待完善 |
| E | `GET/PUT /api/users/me` `GET /api/favorites` `GET /api/history` `GET /api/notifications` `POST /api/shops` 等 | Phase 6 |
| F | `POST /api/reports` `GET /api/admin/*` `POST /api/admin/*` 等 | Phase 7 |

> **对联调建议**：每个 Phase 开始前先用 Swagger (`/docs`) 确认该批次接口可用。后端未就绪时前端用 mock 数据先行开发 UI。
