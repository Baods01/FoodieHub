# 09 — 分享店铺设计规格

> 用户主动上传新店铺的入口页。必填项极简，封面图可选，无主观性描述字段。

---

## 页面定位

用户从计划中的快捷入口（如首页浮动按钮、个人中心「我上传的店铺」区域）到达，或直接访问 `/upload-shop`。核心目标是**用最低的门槛让用户完成店铺信息提交，创建即上线**。

## 逐区域规格

### 1. 顶部标题

| 项目 | 规格 |
|:---|:---|
| 图标 | `MapPin` lucide-react，`text-orange-500` |
| 标题 | 「分享店铺」`text-xl font-bold` |
| 副标题 | 「分享你发现的宝藏美食店铺」`text-sm text-gray-400` |

### 2. 必填字段

#### 2a. 店铺名称

| 项目 | 规格 |
|:---|:---|
| 组件 | `input type="text"` |
| 校验 | 非空，`max_length=100` |
| 错误态 | 红色边框 + 「请输入店铺名称」 |
| placeholder | 「例如：陈记糖水铺」 |

#### 2b. 品类 + 区域（同一行 `grid-cols-2`）

| 项目 | 规格 |
|:---|:---|
| 组件 | `Listbox` 下拉（复用 `FilterDropdown` 样式） |
| 数据来源 | 品类字典：`local_cuisine` / `hotpot` / `barbecue` / `western_food` / `snacks` / `specialty` / `drinks` / `desserts` |
| 区域字典 | `nei_taisan` / `nei_huashan` / `nei_qilin` / `nei_liuyi` / `zhuxiaoqu` / `wai_outside` |
| 自适应 | 品类和区域的选项从字典 API 动态获取（详见后端 `dict_data` 表） |
| 校验 | 两项均必选 |
| 错误态 | 「请选择品类」「请选择区域」 |

#### 2c. 就餐方式（多选 pill 按钮）

| 项目 | 规格 |
|:---|:---|
| 布局 | 3 个 pill 按钮横排，可多选 |
| 选项 | 堂食 `dine_in` / 自取 `pickup` / 外卖 `delivery` |
| 选中态 | `bg-orange-500 text-white border-orange-500` |
| 未选态 | `bg-white text-gray-600 border-gray-200 hover:border-gray-300` |
| 校验 | 至少选一项 |
| 错误态 | 「请至少选择一种就餐方式」 |
| 自适应 | 数据来自字典，自动适应 |

### 3. 封面图片（选填）

| 项目 | 规格 |
|:---|:---|
| 上传区域 | 虚线边框矩形，`h-36`，居中「+ 点击上传封面图片」`text-sm text-gray-400` |
| 上传后 | 显示图片预览 +「重新选择」覆盖按钮 |
| 建议尺寸 | 16:9 |
| 格式 | JPG/PNG/WebP，最大 2MB |
| 实现 | 使用隐藏 `<input type="file">` 触发 |

### 4. 提交按钮

| 项目 | 规格 |
|:---|:---|
| 样式 | 橙渐变 `from-[#FF7E3A] to-[#FF9A5C]`，`w-full py-2.5 rounded-xl` |
| 文字 | 📍 `MapPin` 图标 +「立即分享」|
| 加载态 | 按钮内旋转变色 +「分享中...」|
| hover | `hover:shadow-lg hover:shadow-orange-200 hover:scale-[1.01]` |
| 提交前校验 | 必填项全部通过才可点击 |

### 5. 交互细节

| 交互 | 行为 |
|:---|:---|
| 品类切换 | 选择后右侧区域下拉自动可用 |
| 图片上传 | 点击区域 → 文件选择 → 预览 |
| 图片替换 | 已有预览时，再次点击可替换 |
| 提交失败 | Toast「分享失败，请重试」，保留已填数据 |
| 提交成功 | Toast「店铺分享成功！」+ 跳转新店铺详情页 |

## 数据流

```
页面加载
  ↓
shopApi.fetchDictData('category') → 品类选项
shopApi.fetchDictData('location_type') → 区域选项
  ↓
用户填写 → 客户端校验
  ↓
shopApi.createShop({ name, dict_data_codes, dining_methods, coverImage? })
  ↓ 成功 → navigate(`/shop/${newId}`)
  ↓ 失败 → Toast 错误消息
```

---

## 接口依赖

| 方法 | 路径 | 说明 |
|:---|:---|:---|
| GET | `/api/dict/data?type=category` | 获取品类字典（创建时填充下拉） |
| GET | `/api/dict/data?type=location_type` | 获取区域字典 |
| POST | `/api/shops` | 创建店铺，body: `{ name, dict_data_codes, dining_methods }` |
