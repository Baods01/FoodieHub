# 08 — 消息通知设计规格

> 参照 B 站消息中心风格。左侧消息分类导航 + 右侧消息列表。

---

## 页面定位

用户从 Header 下拉菜单 →「消息通知」到达，或直接访问 `/notifications`。聚合所有与用户相关的互动消息：评论回复、点赞通知、系统公告。

## 逐区域规格

### 1. 整体容器

| 项目 | 规格 |
|:---|:---|
| 布局 | flex row，左侧导航 `w-56` + 右侧内容区 `flex-1` |
| 容器 | 一张大 `SectionCard`，`bg-white rounded-xl shadow-sm`，无内边距 |
| 分隔 | 左右区域用 `border-r border-gray-100` 分隔 |

### 2. 左侧导航栏

| 项目 | 规格 |
|:---|:---|
| 宽度 | `w-56 flex-shrink-0` |
| 顶部标题 | 🔔 `Bell` 图标 +「消息通知」`text-base font-bold`，`p-4 pb-2` |
| 分类列表 | 三个分类按钮纵向排列 |
| 分类按钮 | `w-full text-left px-4 py-2.5 text-sm flex items-center gap-2` |
| 选中态 | `bg-orange-50 text-orange-600 font-medium border-r-2 border-orange-400` |
| 未选态 | `text-gray-600 hover:bg-gray-50` |
| 未读计数 | 每个分类右侧显示未读数（橙色小圆点或数字badge） |
| 底部总览 | 分隔线后显示数量统计小卡片 |

#### 分类列表

| 分类 | 图标 | 说明 |
|:---|:---|:---|
| 回复我的 | `MessageSquare` `text-blue-500` | 评论/回答被回复 |
| 收到的赞 | `ThumbsUp` `text-red-500` | 评论/回答被点赞 |
| 系统通知 | `Megaphone` `text-orange-500` | 管理员发布的公告 |

### 3. 右侧消息列表

| 项目 | 规格 |
|:---|:---|
| 顶部 | 当前分类名称 `text-base font-bold` + 计数 `text-sm text-gray-400`，`p-4 pb-0` |
| 列表 | `divide-y divide-gray-100` |
| 每行消息 | `flex items-start gap-3 px-4 py-3.5 hover:bg-gray-50/50 cursor-pointer transition-colors` |

#### 每行消息组成

```
┌────────────────────────────────────────────────────┐
│  ○  [图标]  描述文字                      时间     │
│                                                 │
│  未读行：左侧橙色小圆点（w-2 h-2 rounded-full    │
│          bg-orange-400 flex-shrink-0 mt-1.5）    │
│  已读行：无圆点                                  │
│  图标：按消息类型区分                            │
│  描述：text-sm text-gray-700                     │
│  时间：text-xs text-gray-400 flex-shrink-0       │
└────────────────────────────────────────────────────┘
```

### 4. 消息类型定制

#### 类型 A：「回复我的」

| 项目 | 规格 |
|:---|:---|
| 图标 | `MessageSquare` `text-blue-500`，`w-8 h-8 rounded-full bg-blue-50 flex items-center justify-center` |
| 描述 | 「XX 回复了你的评论」（仅来源，不展示内容摘要） |
| 来源 | `reply_comment` 消息中 `sender` 的用户名 |
| 点击行为 | 标记已读 + `navigate(/shop/:id)` 跳转到对应店铺详情页 |
| 关联 | 通过 `related_entity_id` 获取店铺 ID |

#### 类型 B：「收到的赞」

| 项目 | 规格 |
|:---|:---|
| 图标 | `ThumbsUp` `text-red-500`，`w-8 h-8 rounded-full bg-red-50 flex items-center justify-center` |
| 描述 | 「XX 赞了你的评论」（仅来源，不展示内容摘要） |
| 点击行为 | 标记已读 + `navigate(/shop/:id)` 跳转到对应店铺详情页（与你那条评论所在店铺一致） |

#### 类型 C：「系统通知」

| 项目 | 规格 |
|:---|:---|
| 图标 | `Megaphone` `text-orange-500`，`w-8 h-8 rounded-full bg-orange-50 flex items-center justify-center` |
| 描述 | 公告标题（`font-medium`）+ 内容摘要 `line-clamp-1` |
| 点击行为 | 弹出非全屏 `Dialog` 展示公告完整内容（标题 + 正文 + 发布时间） |
| 弹窗 | 宽度 `max-w-lg`，标题 `text-base font-bold`，正文 `text-sm text-gray-600 whitespace-pre-wrap` |

### 5. 空态/加载/错误

| 状态 | 处理 |
|:---|:---|
| 列表加载中 | 4 行骨架屏（左侧圆 + 右侧文字条） |
| 某分类无消息 | 居中显示「暂无xx消息」|
| 全部无消息 | 页面居中「暂无消息」+ 🔔 大图标 |
| 加载失败 | `ErrorState` 重试 |
| 公告未读标记 | 同普通消息未读逻辑，点击弹窗查看后标记已读 |

### 6. 交互汇总

| 交互 | 行为 |
|:---|:---|
| 切换分类 | 右侧列表平滑切换（请求对应类型数据） |
| 点击消息「回复我的/收到的赞」| 标记已读 + 跳转店铺详情 |
| 点击消息「系统通知」| 弹出 Dialog 展示全文 + 标记已读 |
| 橙色圆点 | 仅未读消息显示 |
| 全部已读 | 暂不实现（保留未来） |

## 数据流

```
页面加载
  ↓
notificationsApi.fetchNotifications()
  ↓
按 type 分组成三个列表 → 左侧显示各分类计数
  ↓
默认选中「回复我的」→ 渲染对应列表
  ↓
切换分类 → 切换渲染
  ↓
点击消息 → 标记已读（调 API）+ 跳转/弹窗
```

---

## 接口依赖

| 方法 | 路径 | 说明 |
|:---|:---|:---|
| GET | `/api/notifications` | 获取全部消息，返回 `Notification[]` |
| POST | `/api/notifications/:id/read` | 标记单条消息为已读 |
