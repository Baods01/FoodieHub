# 05 — 个人中心设计规格

> 用户个人信息展示与动态时间线聚合页。

---

## 页面定位

用户从 Header 下拉菜单 →「个人中心」到达，或直接访问 `/profile`。核心目标是展示用户身份信息和近期在平台上的活动记录，让其他用户快速了解该用户的参与度。

### 用户场景

| 场景 | 来源 |
|:---|:---|
| 查看个人信息 | 自己或他人访问个人中心主页 |
| 了解用户活跃度 | 查看其最近评价、收藏、上传等动态 |
| 快速跳转 | 点击动态条目进入对应店铺详情页 |

## 逐区域规格

### 1. 个人信息卡

| 项目 | 规格 |
|:---|:---|
| 头像 | `w-24 h-24 rounded-full`，橙渐变 `#FF7E3A → #FF9A5C`，首字白色 |
| 有头像 URL | `object-cover` 显示图片 |
| 无头像 | 显示 `userName.charAt(0)` 首字，`text-3xl font-bold text-white` |
| 用户名 | `text-xl font-bold text-gray-800` |
| 联系方式 | `text-sm text-gray-400`，邮箱 + · + 手机号（中间 4 位 `****` 脱敏） |
| 简介 | `text-sm text-gray-500`，默认「分享让美食更有温度」 |
| 编辑按钮 | `rounded-xl border border-orange-200 text-orange-500`，含 `Edit3` 图标，暂不做编辑功能 |

### 2. 个人动态卡

| 项目 | 规格 |
|:---|:---|
| 标题 | 「个人动态」`text-base font-bold`，左侧 `border-l-[3px] border-orange-400` 橙色装饰条 |
| 每行布局 | 图标(28px) → 描述文字 → 右侧相对时间，flex row |
| 图标 | 按类型显示不同 `lucide-react` 图标，各有主题色 |
| 描述 | `text-sm text-gray-700`，「**用户名** 动作 对象」，用户名加粗 |
| 评分动态 | 额外显示星星行（`fill-yellow-400`） |
| 时间 | 右侧 `text-xs text-gray-400`，格式化相对时间（x小时前/x天前/x月前） |
| 点击 | 整行可点击，`navigate(/shop/:id)` 到店铺详情 |
| 悬停 | `hover:bg-orange-50/50 transition-colors` |
| 加载中 | 3 条 skeleton 脉冲动画 |
| 空状态 | 「还没有动态，开始探索校园美食吧」居中文字 |

### 动态类型映射

| type | 图标 | 颜色 | 描述模板 |
|:---|:---|:---|:---|
| `rating` | `Star` | `text-yellow-500` | 评分了「店名」 |
| `comment` | `MessageSquare` | `text-blue-500` | 评论了「店名」 |
| `favorite` | `Heart` | `text-red-500` | 收藏了「店名」 |
| `add_shop` | `MapPin` | `text-orange-500` | 上传了新店铺「店名」 |
| `question` | `HelpCircle` | `text-purple-500` | 提问了「店名」 |

### 相对时间格式化

```
< 1分钟  → 刚刚
< 60分钟 → N分钟前
< 24小时 → N小时前
< 30天   → N天前
≥ 30天   → N个月前
```

## 接口依赖

| 方法 | 路径 | 说明 |
|:---|:---|:---|
| GET | `/users/me/profile` | 获取用户统计信息（头像URL、店铺数等），当前页面未使用暂保留 |
| GET | `/users/:id/activities` | 获取用户动态列表，返回 `Activity[]` |
