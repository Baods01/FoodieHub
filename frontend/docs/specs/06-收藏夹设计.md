# 06 — 收藏夹设计规格

> 用户收藏的店铺聚合页。采用横版卡片布局，每行一张卡片。

---

## 页面定位

用户从 Header 下拉菜单 →「收藏夹」到达，或直接访问 `/favorites`。核心目标是快速浏览和管理已收藏的店铺，支持跳转店铺详情和取消收藏。

## 逐区域规格

### 1. 顶部标题栏

| 项目 | 规格 |
|:---|:---|
| 左侧 | ❤️ `Heart` 图标（`text-red-500`）+「收藏夹」`text-xl font-bold` |
| 右侧 | 排序下拉（默认「按收藏时间」）+「共 N 家」`text-sm text-gray-400` |

### 2. 横版卡片布局

每张卡片水平排列，占满容器宽度：

```
┌────────────┬─────────────────────────────────┐
│            │  店铺名称（text-base font-bold）  │
│   图片区   │                                  │
│            │  品类标签 · 地区标签（flex gap-2）│
│   w-40     │  N条评论 · N人收藏 · N次浏览     │
│   h-28     │  text-sm text-gray-400           │
│   圆角     │                                  │
│   object-  │  [取消收藏]      收藏于 3天前     │
│   cover    │  red hover        text-xs        │
└────────────┴─────────────────────────────────┘
```

| 项目 | 规格 |
|:---|:---|
| 整体 | `bg-white rounded-xl shadow-sm border border-gray-100`，flex row，`overflow-hidden` |
| 点击 | 整张卡片 `cursor-pointer`，点击跳转 `/shop/:id` |
| 悬停 | `hover:shadow-md hover:-translate-y-0.5 transition-all` |

### 3. 图片区（左侧）

| 项目 | 规格 |
|:---|:---|
| 尺寸 | `w-40 h-28 flex-shrink-0`，固定宽高比 |
| 图片适配 | `object-cover`，`rounded-l-xl overflow-hidden` |
| 无图态 | 灰色 `bg-gray-200` + 居中文字「暂无图片」`text-gray-400 text-sm` |

### 4. 信息区（右侧）

| 区域 | 内容 | 规格 |
|:---|:---|:---|
| 右上角 | ⭐ 评分 | `text-sm text-gray-400`，如 `⭐ 4.5` |
| 左上 | 店铺名称 | `text-base font-bold truncate` |
| 第二行 | 品类标签 + 地区标签 | `flex gap-2`，品类 `border-orange-300 text-orange-600`，地区 `border-gray-300 text-gray-500` |
| 第三行 | 冗余统计 | `text-sm text-gray-400`，格式「N条评论 · N人收藏 · N次浏览」，用 `·` 空格分隔 |
| 左下 | [取消收藏] 按钮 | `text-sm text-gray-400 hover:text-red-500 transition-colors` |
| 右下 | 收藏时间 | `text-xs text-gray-400`，格式「收藏于 3天前」 |

### 5. 数据来源

卡片数据使用 `ShopCardData` 已有字段：

| 字段 | 类型 | 来源 |
|:---|:---|:---|
| `name` | string | 店铺名称 |
| `category` | string | 品类标签显示文字 |
| `area` | string | 地区标签显示文字 |
| `rating` | number | ⭐ 评分 |
| `reviewCount` | number | 评论数（冗余统计） |
| `favoriteCount` | number | 收藏数（冗余统计） |
| `viewCount` | number | 浏览数（冗余统计） |
| `coverImage` | string \| null | 店铺封面图 |
| `tags` | string[] | 店铺标签（当前卡片暂不展示，后续可扩展） |

### 6. 取消收藏交互

| 项目 | 规格 |
|:---|:---|
| 触发 | 点击卡片左下角的「取消收藏」文字按钮 |
| 乐观更新 | 立即从列表中移除该张卡片（整行消失），不等待 API |
| 失败回滚 | API 报错 → 卡片重新出现 + Toast「操作失败，请重试」 |
| 二次确认 | 不做弹窗确认（轻量操作，降低摩擦） |

### 7. 排序

| 项目 | 规格 |
|:---|:---|
| 默认 | 按收藏时间倒序（最新的在最前） |
| 排序下拉 | 复用 `SortDropdown` 组件，选项：收藏时间（默认）、评分最高、名称 A-Z |
| 切换时 | 本地排序，不重新请求接口 |

### 8. 边界状态

| 状态 | 处理 |
|:---|:---|
| 加载中 | 3 行横版骨架屏（左侧 w-40 h-28 灰块 + 右侧文字灰条），脉冲动画 |
| 空收藏 | `Heart` 大图标（`text-gray-200 size-12`）+「还没有收藏任何店铺」+「去发现美食 →」链接首页 |
| 加载失败 | `ErrorState` 重试按钮 |
| 取消收藏失败 | Toast「操作失败，请重试」+ 卡片回滚出现 |

---

## 数据流

```
页面加载
  ↓
favoritesApi.fetchFavorites()
  ↓ 成功
setFavorites(list) → 渲染横版卡片列表
  ↓ 失败
ErrorState

点击「取消收藏」
  ↓
乐观：setFavorites(prev => prev.filter(...))
  ↓
favoritesApi.removeFavorite(shopId)
  ↓ 成功 → 保持移除
  ↓ 失败 → 回滚 + Toast「操作失败，请重试」
```

---

## 接口依赖

| 方法 | 路径 | 说明 |
|:---|:---|:---|
| GET | `/api/favorites` | 获取当前用户收藏列表，返回 `ShopCardData[]`（含收藏时间 `favoritedAt`） |
| POST | `/api/shops/:id/favorite` | 切换收藏状态（已存在→取消，不存在→添加） |
