# 03 — 登录设计规格

> 全站用户认证入口，双栏非对称布局。左品牌展示 + 右表单。

---

## 页面定位

用户在点击导航栏「登录/注册」或进行需要登录的操作（收藏、评论、评分）被拦截后到达。核心目标是**最低摩擦完成认证**，让用户尽快回到浏览流程。

### 用户场景

| 场景 | 来源 |
|:---|:---|
| 主动登录 | 点击 Header 右侧「登录/注册」 |
| 被动拦截 | 游客在详情页点击收藏/评论/评分 → `LoginPromptModal` →「去登录」 |
| 注册后跳转 | 注册成功 → `/login?registered=1` |

## 逐区域规格

### 1. 顶部导航栏

复用全局 `Header` 组件，不单独定制。

### 2. 登录卡片（居中）

| 项目 | 规格 |
|:---|:---|
| 宽度 | `max-w-[840px]` |
| 圆角 | `rounded-2xl`（16px） |
| 阴影 | 双层阴影：`0_2px_8px_rgba(0,0,0,0.04) + 0_8px_32px_rgba(0,0,0,0.10)` |
| 入场动画 | `animate-fade-slide-up`（0.45s，translateY(24px) → 0） |

### 3. 左品牌区（~35%）

| 项目 | 规格 |
|:---|:---|
| 背景 | `from-[#FFE8D6] to-[#FFDCC4]` 暖橙渐变 |
| 品牌圆 | `w-20 h-20` 圆形，`to-[#FF9A5C]` 橙渐变，内含大字「食」 |
| 标题 | 「食探社」text-2xl font-bold |
| 副标题 | 「华农校园美食社区」text-sm text-gray-500 |
| 特点列表 | 3 条，用 `w-1 h-1 rounded-full bg-orange-300` 圆点装饰，无 emoji |

### 4. 右表单区（~65%）

#### 4a. 登录方式切换器

| 项目 | 规格 |
|:---|:---|
| 布局 | 3 个圆形图标按钮横排 + 右侧文字提示 |
| 图标 | `User`（用户名）/ `Smartphone`（手机号）/ `Mail`（邮箱）— 来自 lucide-react |
| 选中态 | `border-orange-400 bg-orange-50 text-orange-500` |
| 未选态 | `border-gray-200 bg-white text-gray-400` |
| 尺寸 | 每个 `w-10 h-10`，`rounded-full` |
| 切换行为 | 点击切换 → 清空输入值 → 清除错误提示 → 更新 label/placeholder/校验规则 |
| 文字提示 | 当前模式显示「使用xxx登录」 |

#### 4b. 账号输入框

| 项目 | 用户名模式 | 手机号模式 | 邮箱模式 |
|:---|:---|:---|:---|
| Label | 用户名 | 手机号 | 邮箱 |
| Placeholder | 请输入用户名 | 请输入11位手机号 | 请输入邮箱地址 |
| 左侧图标 | `User` | `Smartphone` | `Mail` |
| 校验 | 基本非空 | `/^1[3-9]\d{9}$/` | email regex |
| type | text | tel | email |

输入框样式：`rounded-xl、py-2.5、pl-10（图标位）、focus:ring-2 focus:ring-orange-400、hover:border-gray-300`。错误态：`border-red-400 bg-red-50`，错误信息 `animate-fade-in` 出现。

#### 4c. 密码输入框

| 项目 | 规格 |
|:---|:---|
| type | `password`，带 👁 切换显示/隐藏（`Eye/EyeOff` 图标） |
| placeholder | 请输入密码 |
| 校验 | 基本非空 |
| 错误态 | 统一显示「账号或密码错误」（后端返回，不区分用户名/密码哪个错了） |

#### 4d. 登录按钮

| 项目 | 规格 |
|:---|:---|
| 背景 | `from-[#FF7E3A] to-[#FF9A5C]` 橙渐变 |
| 悬停 | `hover:shadow-lg hover:shadow-orange-200 hover:scale-[1.01]` |
| 点击 | `active:scale-[0.99]` |
| 加载态 | 按钮内显示圆形旋转动画 +「登录中...」，`disabled` 禁止重复提交 |

#### 4e. 底部链接

| 项目 | 规格 |
|:---|:---|
| 分隔线 | `border-t border-gray-100`，上方 `mt-8 pt-6` |
| 文案 | 「没有账号？**立即注册**」→ `<Link to="/register">` |

## 数据流

```
用户输入 → 客户端校验（格式/非空）
  ↓ 通过
LoginPage → loginApi(account, password, loginMode)
  ↓ 成功
authStore.login(token, user) → localStorage 存 token
  ↓
navigate('/') → Header 自动切换为已登录态
  ↓ 失败
显示红色错误提示（不区分是账号错还是密码错，防枚举）
```

---

## 交互细节

| 交互 | 行为 |
|:---|:---|
| 回车提交 | 密码框按 Enter 触发表单提交 |
| 模式切换 | 清空输入 + 清除错误 + label/placeholder 即时变化，无页面刷新 |
| 输入框聚焦 | orange 光圈 `focus:ring-2 focus:ring-orange-400` |
| 错误出现 | `animate-fade-in` 0.3s 淡入，不突兀 |
| 卡片入场 | 首次加载 `animate-fade-slide-up` 0.45s |

---

## 边界状态

| 状态 | 处理 |
|:---|:---|
| 空字段提交 | 红色文字提示「请输入xxx」 |
| 格式错误 | 手机号/邮箱实时校验，提交时二次校验 |
| 账号或密码错误 | 密码框下方显示「账号或密码错误」 |
| 请求中 | 按钮 loading 态，禁止重复点击 |
| 网络异常（后续） | Mock 模拟 600ms 延迟，真实 API 接入后可超时提示 |
| 已登录访问 | 不拦截，直接显示登录页（允许切换账号） |

---

## 接口依赖

| 方法 | 路径 | 说明 |
|:---|:---|:---|
| POST | `/users/login` | OAuth2 表单登录，body: `username` + `password`，返回 `{access_token, token_type}` |
| GET | `/users/me` | 页面初始化时调用，从 localStorage token 恢复用户信息 |
