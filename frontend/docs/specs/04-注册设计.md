# 04 — 注册设计规格

> 全站用户注册入口，与登录页同款双栏非对称布局。

---

## 页面定位

用户从登录页底部「没有账号？**立即注册**」跳转到达，或在地址栏直接访问 `/register`。核心目标是用最清晰的引导让用户完成注册，然后回到登录流程。

---

## 逐区域规格

### 1. 顶部导航栏

复用全局 `Header` 组件，不单独定制。

### 2. 注册卡片（居中）

与登录页同款卡片：`max-w-[840px]`、`rounded-2xl`、双层阴影、`animate-fade-slide-up` 入场动画。

### 3. 左品牌区（~35%）

与登录页同款暖橙渐变背景（`from-[#FFE8D6] to-[#FFDCC4]`），但文案不同：

| 元素 | 内容 |
|:---|:---|
| 品牌圆 | 橙渐变，大字「食」 |
| 主标题 | 「加入我们」 |
| 副标题 | 「成为校园美食侦探」 |
| 特点列表 | 分享你发现的宝藏店铺 / 点评问答，和同学真实互动 / 收藏美食，随时查阅 |

### 4. 右表单区（~65%）

#### 4a. 标题区

| 项目 | 规格 |
|:---|:---|
| 主标题 | 「注册」text-xl font-bold |
| 副标题 | 「创建账号，开始探索校园美食」text-sm text-gray-400 |

#### 4b. 用户名

| 项目 | 规格 |
|:---|:---|
| 校验 | 2-50 字符，仅允许字母/数字/下划线/中文 `/^[\w\u4e00-\u9fa5]+$/` |
| 错误提示 | 「用户名至少 2 个字符」「只能包含字母、数字、下划线和中文」「该用户名已被注册」 |

#### 4c. 手机号 + 邮箱（同一行 `grid-cols-2`）

| 项目 | 校验 |
|:---|:---|
| 手机号 | `/^1[3-9]\d{9}$/`，错误提示「手机号格式不正确」 |
| 邮箱 | email regex，错误提示「邮箱格式不正确」 |

#### 4d. 密码 + 确认密码（同一行 `grid-cols-2`）

| 项目 | 规格 |
|:---|:---|
| 密码 | 最少 6 位，带 👁 显示/隐藏切换 |
| 确认密码 | 必须与密码一致，错误提示「两次输入的密码不一致」 |
| 密码一致性 | 提交时校验 `password === confirmPwd` |

#### 4e. 注册按钮

| 项目 | 规格 |
|:---|:---|
| 样式 | 同登录页橙渐变按钮 |
| 加载态 | 按钮内旋转变色 +「注册中...」 |
| 提交失败 | 根据错误消息内容智能映射到对应字段（用户名/手机号/邮箱） |

#### 4f. 底部链接

| 项目 | 规格 |
|:---|:---|
| 分隔线 | `border-t border-gray-100` |
| 文案 | 「已有账号？**立即登录**」→ `<Link to="/login">` |

---

## 数据流

```
用户输入 → 客户端校验（5 个字段）
  ↓ 通过
RegisterPage → registerApi(username, password, phone, email)
  ↓ 成功
navigate('/login?registered=1')  ← 回到登录页
  ↓ 失败
根据错误消息关键词映射到具体字段显示红色提示
```

---

## 交互细节

| 交互 | 行为 |
|:---|:---|
| 所有输入框 | `rounded-xl`，聚焦 `focus:ring-2 focus:ring-orange-400` |
| 错误出现 | `animate-fade-in` 淡入 |
| 按钮悬停 | `hover:scale-[1.01]` 微放大 + 橙光阴影 |
| 按钮点击 | `active:scale-[0.99]` 微缩小反馈 |
| 手机号+邮箱同行 | 小屏自动 `grid-cols-1` 折行 |
| 密码+确认同行 | 同理 |
| 回车提交 | 任意输入框按 Enter 触发 |

---

## 边界状态

| 状态 | 处理 |
|:---|:---|
| 空字段提交 | 各字段显示「请输入xxx」 |
| 用户名已存在 | 用户名下方显示「该用户名已被注册」 |
| 手机号格式错 | 手机号下方显示「手机号格式不正确」 |
| 邮箱格式错 | 邮箱下方显示「邮箱格式不正确」 |
| 密码不一致 | 确认密码下方显示「两次输入的密码不一致」 |
| 请求中 | 按钮 loading |
| 注册成功 | 跳转 `/login` （后端未做自动登录） |

## 接口依赖

| 方法 | 路径 | 说明 |

| POST | `/users/register` | body: `{ username, password, phone, email }`，返回 `UserResponse` |
