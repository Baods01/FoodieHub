# 02 — 店铺详情页设计规格

> 全站信息密度最高的页面。用户在主页点击卡片后到达，承载店铺完整信息、评分、评论、问答等核心交互。

---

## 页面定位

用户在主页/搜索/收藏夹点击店铺卡片后到达，核心动线是**查看评分 → 浏览评论/问答做消费决策 → （如已消费）贡献评分/评论/提问**。页面是社区信息沉淀的最终展示端。

---

## 页面布局（从上到下）

```
┌─────────────────────────────────┐
│  导航栏（复用 Header + 返回按钮）  │
├─────────────────────────────────┤
│  轮播图（自动轮播 + 圆点指示器）   │
├─────────────────────────────────┤
│  店铺信息 + 标签              │
│  [反馈按钮]                     │
├─────────────────────────────────┤
│  你的评分：[★ ★ ★ ★ ★]         │
│  ⭐ 4.3  │  评分分布条形图        │
│  共 32 人评分                     │
├─────────────────────────────────┤
│  评论区                           │
│  └ 发布框（常驻顶部）             │
│  └ 评论列表（点赞数降序）          │
│    └ 回复：点击展开回复框           │
├─────────────────────────────────┤
│  问答区                           │
│  └ 提问框（常驻顶部）             │
│  └ 问答列表（按最新回答排序）      │
│    └ 回复逻辑同评论区              │
├─────────────────────────────────┤
│  菜单（卡片横向滚动）              │

├─────────────────────────────────┤
│  店铺相册                         │
│  [上传图片]                       │
│  └ 最多 3 行网格，超出滚动按钮     │
```

## 逐区域详细规格

### 1. 导航栏

复用主页的 `Header` 组件，但左侧增加返回按钮 `<`：

| 项目 | 规格 |
|:---|:---|
| 左侧 | 返回箭头图标 + "食探社" Logo，点击返回上一页 |
| 右侧 | 同主页：未登录→"登录/注册"，已登录→头像+昵称 |
| 背景 | 白色，底部 1px 浅灰分割线 |

### 2. 轮播图

**图片源**：店铺相册中的所有图片。若店铺相册为空，显示单张灰色占位图 + "暂无店铺照片"。

**交互**：

| 项目 | 规格 |
|:---|:---|
| 切换方式 | 自动轮播 + 用户可点击底部圆点跳转 |
| 轮播间隔 | 3 秒 |
| 指示器 | 底部居中一排小圆点，实心=当前，空心=其他 |
| 图片适配 | object-cover，等比例填充容器 |
| 高度 | 固定 320px（h-80），响应式可缩至 200px |
| 过渡 | 淡入淡出或滑动过渡，transition 0.3s |
| 无图时 | 灰色 bg-gray-200 + 居中文字"暂无店铺照片" |

### 3. 店铺信息区

| 项目 | 规格 |
|:---|:---|
| 店铺名 | text-2xl font-bold |
| 标签行 | 品类标签（橙色边框）+ 区域标签（灰色边框），flex gap-2 mt-2 |
| 地址 | 纯文本展示，text-sm text-gray-500 mt-1（如有详细地址） |
| 反馈按钮 | 右侧/右下角 "反馈" 文字按钮（text-sm text-gray-400 hover:text-orange-500），点击触发什么操作由后续页面决定（跳转勘误/举报页） |

### 4. 评分区域

**布局**：分两行。

```
第一行：你的评分：[★ ★ ★ ★ ★]           ← 水平居中
第二行：⭐ 4.3  │  ████████░░ 条形图排列  │  共 32 人评分
                              ← 水平一排
```

**评分交互**：

| 用户状态 | 交互 |
|:---|:---|
| **游客** | 点击星星 → 弹出登录引导弹窗"登录后即可评分" |
| **未评分的登录用户** | 点击星星 → 星星高亮至所选星级 → 弹出确认框"确认评为 N 星？" → 点"确认"提交 / 点"取消"恢复 |
| **已评分的登录用户** | 显示"你的评分：★★★"（只读，灰色不可点），保留"你的评分："前缀不变 |

**评分分布**：

| 项目 | 规格 |
|:---|:---|
| 综合评分 | 左侧大号字体（text-4xl font-bold）+ ⭐ 图标，保留一位小数（如 4.3） |
| 分布条 | 右侧纵向排列 5 个条形条（5星→1星从上到下），每个条形用灰色/橙色填充比例 |
| 条形条 | 纯 CSS 横向条形图，宽度表示该星级占比，**不显示人数和百分比数字** |
| 总评人数 | 综合评分下方或分布条区域底部显示 "共 N 人评分"（text-sm text-gray-400） |
| 暂无评分 | 综合评分显示"--"，分布条全灰，"暂无评分"文字替代"共 N 人评分" |

### 5. 评论区（核心功能区 #1）

**布局**：发布框常驻顶部 → 评论列表（分页加载）。

**发布框**（仅登录用户可见，游客显示"登录后即可评论"提示）：

| 项目 | 规格 |
|:---|:---|
| 位置 | 评论区顶部，常驻显示 |
| 控件 | 多行文本输入框 + 图片上传（可选）+ "发布"按钮 |
| 文字 | 1-500 字符，非空校验 |
| 图片 | 最多 3 张，jpg/png，单张 ≤5MB，上传前压缩 |
| 行为 | 点击"发布"→ 按钮显示"发布中..." → 成功后评论插入列表顶部并 Toast "评论已发布"，失败 Toast "发布失败，请重试" |

**评论列表**：

| 项目 | 规格 |
|:---|:---|
| 排序 | 按点赞数降序（点赞数相同按时间倒序） |
| 分页 | 每页 10 条，点击"加载更多"或滚动加载 |
| 单条评论 | 用户头像+昵称 + 时间 + 文字 + 图片（最多3张，网格排列）+ 点赞按钮 + 回复按钮 |
| 点赞 | ♡/♥ 切换，乐观更新（先变心形+1，失败回滚） |

**回复逻辑**：

| 项目 | 规格 |
|:---|:---|
| 触发 | 点击评论底部的"回复"按钮 |
| 展开 | 在该评论下方弹出回复输入框（单行或小多行文本框 + "发送"按钮） |
| 回复列表 | 该评论的所有回复以缩进形式展示在评论下方（1-2 层，不做无限嵌套） |
| 排序 | 按时间正序（最早的在前） |
| 通知 | 被回复者后续获得站内信通知（预留，当前仅 UI） |

### 6. 问答区（核心功能区 #2）

**提问框**（仅登录用户可见）：

| 项目 | 规格 |
|:---|:---|
| 样式 | 同评论区发布框，简化：标题输入框（2-50位） + 内容输入框（5-200位）+ "提问"按钮 |
| 行为 | 点击"提问" → 提问即刻展示在列表顶部，无需审核 |

**问答列表**：

| 项目 | 规格 |
|:---|:---|
| 排序 | 按最新回答时间降序（最近有人回答的排在前面；无回答的按提问时间降序） |
| 单条问题 | 用户头像+昵称 + 问题标题 + 问题内容 + 回答数标签 |
| 回答展开 | 点击问题卡片展开/收起下方的回答列表 |
| 回答列表 | 按时间正序排列 |
| 回复回答 | 每条回答底部有"回复"按钮，逻辑同评论区的回复展开 |

### 7. 菜单区

| 项目 | 规格 |
|:---|:---|
| 布局 | 单行横向滚动（overflow-x-auto），flex gap-3，左右边缘渐隐提示 |
| 卡片大小 | w-32 flex-shrink-0，卡片内图片占主体 |
| 卡片样式 | 图片（100×100 或 4:3 比例 rounded-lg）+ 下方菜品名（text-sm truncate） |
| 无图时 | 灰色占位块 + 菜品名文字 |
| 上传菜单按钮 | 区域右上角"上传菜单"文字按钮（仅登录用户可见，游客隐藏） |
| 点击卡片 | 弹出详情弹窗（Modal）：|
| 弹窗内容 | 多行多列网格（竖向滚动），含菜品名、价格、描述、大图 |
| 弹窗关闭 | 点击遮罩层或 × 按钮关闭 |
| 无菜单 | 显示占位提示"还没有菜单，快来上传吧" + "上传菜单"按钮 |

### 8. 店铺相册区

| 项目 | 规格 |
|:---|:---|
| 布局 | CSS Grid，grid-cols-3 gap-2 |
| 行数限制 | 最多 3 行（9 张图），超出时右侧/底部出现滚动按钮 ◀ ▶ 或"查看全部" |
| 缩略图 | aspect-square object-cover rounded |
| 无图 | 不显示该区域（或显示占位提示"还没有店铺照片"） |
| 上传按钮 | 区域右上角"上传图片"文字按钮（仅登录用户可见） |
| 点击缩略图 | 弹出 Lightbox 弹窗展示原图（全屏或近全屏深色背景 + 居中图片，可左右切换） |

## 游客 vs 登录用户差异总表

| 区域 | 游客 | 登录用户 |
|:---|:---|:---|
| 评分星星 | 可点 → 弹出登录弹窗 | 可点 → 确认框 → 提交 / 已评分则只读 |
| 评论发布框 | 显示"登录后即可评论"灰色提示 | 显示完整发布框 |
| 评论点赞 | 可点 → 弹出登录弹窗 | 可点，乐观更新 |
| 提问框 | 显示"登录后即可提问"灰色提示 | 显示完整提问框 |
| 上传菜单 | 隐藏按钮 | 显示按钮 |
| 上传图片 | 隐藏按钮 | 显示按钮 |
| 反馈按钮 | 显示，点击跳转前先登录 | 显示，直接跳转 |
| 其余区域 | **全部可见，无差异** | — |

## 边界状态

| 状态 | 表现 |
|:---|:---|
| 页面加载中 | 整体骨架屏：封面图块 + 文字条 + 评分条 + 3 条评论骨架 |
| 页面加载失败 | ErrorState "加载失败" + "点击重试"按钮 |
| 店铺不存在/已封禁 | 独立提示页 "该店铺已不存在或已被封禁" + "返回首页" |
| 评论为空 | 评论区显示 "还没有评论，来写第一条吧" |
| 问答为空 | 问答区显示 "还没有提问" |
| 菜单为空 | 菜单区显示占位提示 + 上传按钮 |
| 相册为空 | 相册区不显示（轮播图显示占位图） |
| 提交评论失败 | Toast "发布失败，请重试" |
| 提交评分失败 | Toast "评分失败，请重试"，星星回滚至之前状态 |

## 设计风格规范

| 维度 | 规格 |
|:---|:---|
| 风格 | 现代简约 |
| 背景主色 | 白色 #FFFFFF |
| 品牌高亮色 | 渐变橙色 #FF7E3A → #FF9A5C（按钮、标签、强调文字） |
| 卡片 | 白色圆角卡片，圆角 8px，轻阴影 |
| 头像 | 圆形 border-radius: 50% |
| 间距 | 宽松，各区域间距 24px（space-y-6） |
| 字体 | 系统默认中文字体，正文 14px |

---

## 响应式适配

| 断点 | 调整 |
|:---|:---|
| ≤ 768px | 轮播图高度缩至 200px；评分区域改为纵向排列（综合评分在上，分布条在下）；单行菜单卡片缩小至 w-28；相册改为 grid-cols-2 |
| ≤ 480px | 相册改为 grid-cols-1；发布框宽度满屏 |

## 技术约束

- **组件拆分**：ShopCarousel, ShopInfo, RatingSection, CommentSection, CommentCard, ReplyBox, QASection, MenuSection, MenuCard, MenuDetailModal, AlbumSection, AlbumLightbox — 各自独立
- **评分状态**：用 `useState` 管理交互态（悬停/选中/已确认），提交后才持久化
- **弹窗**：基于 Headless UI `Dialog` 实现评分确认框、菜单详情弹窗、相册 Lightbox
- **乐观更新**：评论点赞用 TanStack Query `useMutation` onMutate/onError 实现
- **轮播**：用 `useState` 管理 currentIndex + `useEffect` 实现自动轮播定时器，清除定时器防内存泄漏
- **无限滚动**：评论区复用 `useInfiniteScroll` hook
- **Mock 数据**：准备 1 条完整店铺详情（含 5 条评论、3 条问答、6 个菜品、6 张相册图）
